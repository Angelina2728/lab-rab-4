Ответы на вопросы
1. Какой подход оказался удобнее и почему?

Tailwind CSS показался удобнее для точной настройки внешнего вида.
Utility-классы позволяют сразу видеть визуальные изменения в HTML, не обращаясь к отдельному CSS-файлу.
Тем не менее, Bootstrap удобен для быстрой сборки: готовые компоненты, сетка, предустановленные стили.

Вывод:

Bootstrap — быстрее для прототипа.

Tailwind — гибче для кастомного дизайна.

2. Пример кода для одного и того же элемента
 Кнопка в Bootstrap:
<a href="#" class="btn btn-primary">Подробнее</a>

 Кнопка в Tailwind CSS:
<a href="#" class="bg-purple-600 hover:bg-purple-700 text-white font-semibold py-2 px-4 rounded transition">
  Подробнее
</a>


Ключевое различие:

В Bootstrap используется компонентный класс (btn, btn-primary), где стиль определяется заранее.

В Tailwind — комбинация утилитарных классов, которые задают цвет, отступы, тень и эффекты прямо в HTML.

3. С какими сложностями вы столкнулись
Bootstrap:

Ограниченные возможности кастомизации без переопределения CSS.

Некоторые готовые компоненты выглядят одинаково без дополнительной стилизации.

Tailwind CSS:

В начале сложно привыкнуть к множеству классов в одном теге.

Нужно понимать систему отступов, цветов и брейкпоинтов.

 Выводы

Обе библиотеки позволяют быстро создать современный адаптивный лендинг.

Bootstrap — оптимален для быстрого прототипирования.

Tailwind CSS — для детальной настройки дизайна и точного контроля над стилями.

В рамках этой работы использование обеих библиотек показало, что выбор зависит от цели:

Если нужно быстро и стандартно — Bootstrap.
Если нужно индивидуально и современно — Tailwind.
